a cheat sheet for HTML forms

by Jersy McKupsberg

Type of Form Tag Description Other Notes
Radio Button <input type = "radio"> A radio button or option button is a graphical control element that allows the user to choose only one of a predefined set of mutually exclusive options. The singular property of a radio button makes it distinct from checkboxes, where the user can select and unselect any number of items. You do not need to add a required attribute for the second radio button. If two radio buttons share the same name, only one needs to have the required attribute.
Checkbox Input <input type = "checkbox"> The checkbox is shown as a square box that is ticked (checked) when activated. Checkboxes are used to let a user select one or more options of a limited number of choices. Tip: Always add the <label> tag for best accessibility practices!
password input type="password" provides a way for the user to securely enter a password. The element is presented as a one-line plain text editor control in which the text is obscured so that it cannot be read, usually by replacing each character with a symbol the type of symbol eg asterisk ("*") or a dot ("•") will vary depending on the user agent and operating system.
Dropdown List <select> and <option> The <select> element is used to create a drop-down list. The <select> element is most often used in a form, to collect user input. The <name> attribute is needed to reference the form data after the form is submitted (if you omit the <name> attribute, no data from the drop-down list will be submitted). The <id> attribute is needed to associate the drop-down list with a label. The <option> tags inside the <select> element define the available options in the drop-down list. Tip: Always add the <label> tag for best accessibility practices!
Datalist Input <input type="text"> The <datalist> is used with an <input type="text"> element. The <input> creates a text field that users can type into and filter options from the <datalist>. basically a dropdown list plus a little text field where a user can add/submit their own value
Text Area <textarea> The <textarea> tag defines a multi-line text input control. The <textarea> element can be used to collect textual user inputs (comments, reviews, etc). A text area can hold an unlimited number of characters, and the text renders in a fixed-width font (usually Courier). The size of a text area is specified by the cols and rows attributes (or with CSS). The name attribute is needed to reference the form data after the form is submitted (if you omit the name attribute, no data from the text area will be submitted). The id attribute is needed to associate the text area with a label.

Examples of syntax implentation for each form type

radio button

<form>

<p>What is sum of 1 + 1?</p>
<input type="radio" id="two" name="answer" value="2">
<label for="two">2</label>
<input type="radio" id="eleven" name="answer" value="11">
<label for="eleven">11</label>

</form>
